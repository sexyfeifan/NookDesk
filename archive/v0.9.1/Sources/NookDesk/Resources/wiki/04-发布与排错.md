# 04-发布与排错

## 发布前建议检查

1. 项目结构是否正常（package.json、vite.config.ts 等）
2. GitHub Token 是否可用
3. Pages 来源是否为 GitHub Actions
4. Workflow 文件是否存在
5. 构建是否成功（npm run build）

## 推荐发布顺序

1. 写作页保存内容
2. 发布页查看预检清单
3. 修复所有预检错误
4. 点击「一键发布」
5. 查看最新 GitHub Actions 状态

## 常见问题

### 未配置远程地址
请在设置中填写 GitHub 仓库地址。

### Token 无效
请检查 Token 权限是否包含 repo/workflow。

### Workflow 缺失
点击「生成 Pages Workflow」自动创建。

### Pages 来源错误
点击「修复 Pages 来源」切换到 GitHub Actions。

### 构建失败
检查项目依赖是否安装（npm ci），配置是否正确。
