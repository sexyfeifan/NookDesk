# 04-发布与排错

## 发布前建议检查

1. Hugo 项目结构是否正常
2. GitHub Token 是否可用
3. Pages 来源是否正常
4. Workflow 是否存在
5. 当前主题是否识别正确
6. 当前工作区是否存在失效引用

## 推荐发布顺序

1. 写作页保存内容
2. 发布页做预检
3. 查看是否有权限或结构错误
4. 再执行发布流程
5. 查看最新 GitHub Actions 状态

## 常见问题

### 页面资源按钮不可用
通常是因为当前内容不是页面包。

### 页面引用为空
通常是因为当前工作区没有已保存内容。

### Pages 检查失败但 Git 推送成功
这通常说明 Pages API 权限与 Git 推送权限不一致。
