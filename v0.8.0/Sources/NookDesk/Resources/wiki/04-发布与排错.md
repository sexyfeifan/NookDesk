# 发布与排错

## 一键发布

切换到「发布」标签页，点击「一键发布」：

1. **检查项目状态** — 验证项目配置是否完整
2. **提交并推送** — 将修改推送到 GitHub
3. **等待部署** — GitHub Actions 自动构建并部署

## 环境状态

发布页面顶部显示：
- 项目路径
- 后端类型（Astro / Hugo / Vite）
- 远程仓库地址
- GitHub Token 状态
- Actions 部署状态

## 辅助操作

- **同步远端** — 拉取远程最新代码
- **仅提交推送** — 不等待部署
- **生成 Workflow** — 自动生成 GitHub Actions 配置

## 常见问题

### 发布失败

检查以下项目：
1. GitHub Token 是否有效
2. 远程仓库地址是否正确
3. 网络连接是否正常
4. GitHub Actions 是否有权限

### 构建失败

到 GitHub 仓库 → Actions 页面查看错误日志。常见原因：
- `package.json` 依赖问题
- Astro 配置错误
- Markdown 文件格式错误

### 文章未显示

1. 确认文章在 `src/content/blog/` 目录下
2. 确认 frontmatter 格式正确
3. 确认 `draft: false`（或未设置 draft）
4. 确认已保存并推送

### 环境诊断

点击「环境诊断」按钮，NookDesk 会检查：
- 项目结构是否完整
- Git 配置是否正确
- 远程仓库是否可达
- GitHub Pages 配置是否正确
